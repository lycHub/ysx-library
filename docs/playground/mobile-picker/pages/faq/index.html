为社么要写模板代码？

插件确实可以设计成在new picker()内部生成默认的模板，但这缺乏样式的定制性，所以在入口文件导入内置的样式，然后在html中写模板是更好的选择。

点击选择无效？

clickToSelect = true后， 内部的click事件是绑在".mobile-picker-view"上的, 也就是".mobile-picker-view-item"的父元素上的，而不是item本身。 然后通过itemClassName(默认"mobile-picker-view-item")的值来锁定具体item。 所以此功能无效的原因可能是：
  1. item原生绑定了click事件并阻止了冒泡
  2. 改变了item的classname并且没有正确设置itemClassName
